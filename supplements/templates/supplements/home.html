{# Design decision: single-click behavior on cells #}
DESIGN DECISION — Click behavior in the matrix
Current choice: Single-click on a cell switches to By College view for that school (so any click on a row navigates away from the matrix). Double-click goes to focus mode.
Alternative: Single-click on a cell does nothing (or highlights it); only clicking the column header navigates to By College. Double-click still → focus. This keeps the matrix stable and browsable without accidentally leaving it.

{# Design decision: empty cell behavior #}
DESIGN DECISION — Clicking an empty matrix cell
Current choice: Empty cells are not clickable — you'd need to go to "By College" and create a new essay manually. This keeps data creation intentional.
Alternative: Clicking an empty cell auto-creates a blank essay for that college + category combination and immediately opens it in focus mode. Much faster workflow, but might create accidental essays that are hard to delete.

{# Design decision: always-visible textarea #}
DESIGN DECISION — Textarea visibility
Current choice: Textareas are always visible and immediately editable — no clicking needed to start writing.
Alternative: Show only the prompt + a word count chip by default; click anywhere on the card to expand the textarea. Cleaner when just reviewing, but adds friction for editing.
{# Design decision: topic tagging UI #}
DESIGN DECISION — Topic tagging interface
Current choice: A compact dropdown on each card lets you assign any existing category (topic). Changes immediately update the matrix.
Alternative: A drag-and-drop or type-ahead tag picker — more discoverable and faster when there are many categories, but much more complex to build.
